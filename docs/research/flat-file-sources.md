# Research: flat files as migration sources (dumps, CSV/TSV/JSON, Parquet)

Date: 2026-07-04 · Trigger: operator idea — accept mysqldump / mydumper / `pscale database dump` / pg_dump output, and schema-less formats (CSV, TSV, JSON), as `--source` inputs. · Method: format-spec survey (authoritative docs + the planetscale/cli dumper source) crossed with sluice's existing composable machinery. Status: research + phased recommendation; roadmap item 55 tracks the backlog. Nothing implemented.

## Headline conclusions

1. **The answer is per-format, not uniform.** The formats split cleanly into: regular-and-per-table (mydumper, `pscale database dump` — which is VERIFIED mydumper-compatible per the planetscale/cli source, and mysqldump `--tab`) where a native reader is sound; schema-less (CSV/TSV/NDJSON) where sluice's existing stage-into-SQLite + `--infer-types` machinery fits verbatim; typed-and-self-describing (Parquet) where a native reader is sound and *needs no inference*; and full-dialect SQL dumps (plain mysqldump/pg_dump `.sql`, pg_dump `-Fc`/tar) where native parsing would violate the IR-first no-DDL-grammar tenet or reverse-engineer a privately-versioned format — those get loud refusal + a documented scratch-server recipe.
2. **sluice already owns both halves of the hard machinery.** The ADR-0130 dump-materialize path (`sqlite/dump.go` — streamed statement-splitting into a temp modernc DB, never whole-in-RAM), the ADR-0145 stage-local pattern, `--infer-types` (ADR-0144 — validated promotion, not guessing), the d1 engine's source-only registry shape, and `TableFilter` compose into most of what phases 1–2 need. sluice's own backup format stays a parallel contract (typed IR schema + SHA256 chain that foreign formats can't meet), not subsumed.
3. **DuckDB-as-staging is disqualified on dependency posture** despite genuinely better types (its Go driver requires CGO; sluice is deliberately CGO_ENABLED=0 — the modernc-over-mattn decision). SQLite staging + infer-types reaches most of the same fidelity for schema-less input.
4. **Demand honesty:** strongest signal is provider-only-gives-dumps (PlanetScale's own export *is* mydumper format — phase 1 doubles as PlanetScale-story investment) and air-gapped migrations; CSV onboarding is a larger but less sluice-shaped audience; Parquet-as-source is the rare direction (the analytics research covers the popular inverse). Zero user requests exist yet — phasing starts where inferred demand and existing-machinery reuse overlap.

## Format facts that shape the design (verified)

- **mydumper/pscale dirs:** `metadata` (binlog pos/GTID — a future CDC-handoff hook), per-table `*-schema.sql` (ONE CREATE TABLE each — a bounded parse, not full SQL), data chunks `db.table.NNNNN.sql` containing only extended INSERTs (~1 MB statements), optional gzip/zstd. pscale's writer backslash-escapes binary with **no hex-blob** — binary fidelity rides entirely on faithful escape decoding (silent-loss pin territory).
- **mysqldump plain `.sql`:** extended INSERTs inside a charset-interpreted stream with versioned-conditional `/*!40101 */` blocks; `--hex-blob` is silently ignored under `--tab`; correct parsing requires MySQL string-literal lexing. DDL half = full grammar → tenet violation.
- **pg_dump:** COPY text blocks are precisely specified and line-streamable (the data half is trivial); the DDL half is full PG grammar; `-Fc`/tar are explicitly private (`pg_backup_archiver.h`: "NOT intended that these routines be called by any dumper directly"); `-Fd`'s per-table `.dat.gz` are COPY text but the table mapping lives in binary `toc.dat`.
- **CSV:** RFC 4180 has NO null representation — NULL-vs-empty-string is pure producer convention, the #1 silent-loss class for CSV ingest; must be an explicit operator flag, never sniffed. **JSON:** RFC 8259 interop is binary64 — int64 > 2^53 silently rounds through generic parsers (the exact class the D1 engine's raw-text number handling defeats); NDJSON streams, single-array JSON doesn't. **Parquet:** typed Thrift footer schema (DECIMAL, UUID, signed/unsigned INT, temporal-with-UTC-flag), streamable by row group at 100 GB; Go deps already vetted CGO-free in `apache-arrow-findings.md`.
- **Prior art:** pgloader — the most successful file-loader — deliberately parses **no** SQL dump files (live connections + delimited files only). Nobody orchestrates scratch-server dump replay as a product feature; it's ops folklore. That's either sluice's opportunity or a warning — treated as demand-gated.

## Recommended phases (roadmap item 55)

- **Phase 1 (M): `mydumper` source engine** — native reader for mydumper/pscale-dump directories. Source-only registry entry (the d1 shape: CDCNone, ErrNotImplemented writers), SchemaReader parsing per-table `*-schema.sql` (single MySQL CREATE TABLE — the contained DDL-parse exception needs its own ADR blessing against the IR-first tenet; ADR-0133 is the precedent), RowReader streaming INSERT-VALUES chunks + gz/zstd. Silent-loss pins per Bug-74 discipline: every value family × escape shape (backslash-escaped binary, `0x…` hex, `SET NAMES` charset, NULL vs `'NULL'`, extended-INSERT tuple lexing) ground-truthed against a real myloader-restored database. Verify = per-chunk rescan for count depth.
- **Phase 2 (S/M): CSV/TSV/NDJSON via stage-into-SQLite** — extend the ADR-0130 sniff; stage as TEXT into a temp `.db`, auto-engage `--infer-types` (the D1 auto-stage template). Explicit flags, never guessed: `--csv-null`, header presence, encoding (refuse non-UTF-8 loudly); JSON numbers carried as raw text (the D1 int64 lesson).
- **Phase 3 (S): loud refusal + recipes for plain `.sql` / `-Fc`** — detect `-- MySQL dump` / `PGDMP` / `-- PostgreSQL database dump` signatures at open; refuse naming the scratch-server-replay recipe (docker run --rm + native restore tool + `sluice migrate`). Converts a confusing failure into guidance and collects real demand signals for any future `--scratch <dsn>` orchestration (deferred).
- **Phase 4 (M, demand-gated): Parquet source** — footer schema → IR directly; reuse the parquet-go vetting. Gotchas: UTC-flag-only timestamps, unsigned INT(64), DECIMAL over FIXED_LEN_BYTE_ARRAY.
- **Non-goals (explicit):** parsing pg_dump plain DDL or `-Fc`/`toc.dat`; a DuckDB dependency; subsuming `sluice restore`/the backup format.
